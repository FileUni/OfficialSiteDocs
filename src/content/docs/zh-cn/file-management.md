# 访问方式与文件操作

本页只描述当前项目里已经能从源码和入口层确认的访问方式与文件操作。

## WebUI

内嵌前端通过下面的路径提供：

```text
/
```

WebUI 是当前最主要的人机交互入口，负责：

- 浏览文件与目录
- 上传、下载
- 搜索与预览
- 最近访问、收藏、回收站、分享等视图
- 在浏览器中调用后端 API

## HTTP API

服务端的 REST 风格接口位于：

```text
/api/v1
```

如果要查看接口定义或生成客户端类型，当前项目还会暴露：

```text
/api/v1/openapi.json
```

这是当前对外最准确的接口面描述。

## 协议访问

按配置启用后，FileUni 还可以提供这些标准协议入口：

- WebDAV：挂载在 `/webdav`
- S3：监听配置中指定的 S3 端口
- FTP：监听配置中指定的 FTP 端口
- SFTP：监听配置中指定的 SFTP 端口

服务启动成功后，终端会打印当前 HTTP 地址，以及 S3、FTP、SFTP 是否启用。

## 已确认存在的文件操作

当前 VFS 与前端模块里已经有实现的能力包括：

- 目录浏览
- 文件上传与临时上传处理
- 文件下载
- 移动与复制
- 重命名
- 删除到回收站以及从回收站恢复
- 搜索
- 收藏
- 分享记录与分享筛选
- 压缩包浏览、压缩、解压

## 使用上的说明

这些能力里有一部分会受到配置开关控制。实际部署时：

- 如果协议在配置中被关闭，运行时就不会开放
- 如果某些前端能力被关闭，界面入口也可能不会显示
- 不同 build mode 也可能影响最终打包内容

## 推荐的访问选择

以下场景优先使用 WebUI：

- 想要最直接地浏览和管理文件
- 需要预览能力
- 需要通过现成前端完成管理操作

以下场景更适合使用 WebDAV、S3、FTP、SFTP：

- 需要接入现有工具链
- 需要挂载磁盘或接第三方文件管理器
- 需要脚本化和自动化访问
