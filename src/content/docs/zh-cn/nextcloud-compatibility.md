# Nextcloud 兼容性

本页用于说明 FileUni 在对接 Nextcloud 客户端时的兼容性目标。

除了可运行在低性能硬件、并且在高性能硬件上获得更高吞吐之外，Nextcloud 客户端兼容性也是 FileUni 一个需要明确对外传达的重要特色。

## 兼容目标

FileUni 的目标是，在用户从 Nextcloud 体系迁移时，尽量保持主流 Nextcloud 客户端的使用方式不变。

当前重点覆盖这些兼容方向：

- 基于 WebDAV 的文件同步与远程访问
- 桌面端和移动端常见的文件管理操作
- 收藏、分享、媒体相关访问流程，以及常见元数据交互
- 尽量保持现有 Nextcloud 客户端使用习惯，而不是要求用户重新学习一套客户端逻辑

## 当前兼容范围

当前仓库里已经可以看到这条路线所依赖的基础能力：

- 存储栈中的 WebDAV 服务支持
- WebUI 与 API 已覆盖文件浏览、上传、下载、重命名、移动、复制、删除、最近访问、收藏、回收站等流程
- 前端已包含媒体预览和常见文件预览能力
- 可按部署规模启用的模块化协议服务

这些能力构成了 Nextcloud 兼容路线的基础。实际客户端表现仍然会受到配置、启用模块以及具体 Nextcloud 客户端版本的影响。

## 兼容性清单

| 范围 | 目标 | 状态 |
| --- | --- | --- |
| WebDAV 端点访问 | 兼容 Nextcloud 客户端通过 WebDAV 建立连接 | 已具备基础 |
| 文件管理 | 浏览、上传、下载、重命名、移动、复制、删除 | 当前平台已实现 |
| 收藏 | 让收藏相关流程可兼容 Nextcloud 客户端 | 已有能力基础 |
| 分享 | 让分享相关流程兼容 Nextcloud 客户端 | 兼容目标 |
| 媒体 | 让媒体浏览与预览流程尽量兼容 Nextcloud 风格客户端 | 已有能力基础 |
| Chat | 后续开发计划 | 规划中 |
| Notes | 后续开发计划 | 规划中 |

## 状态标签说明

- 已具备基础：仓库中已经存在这部分所需的协议层或平台层基础。
- 当前平台已实现：FileUni 自身已经具备这项能力，并构成兼容性基础。
- 已有能力基础：相关能力已经存在，后续兼容工作建立在这些能力之上。
- 兼容目标：这是明确的产品方向，但不应理解为所有客户端边缘场景都已经完全覆盖。
- 规划中：属于后续路线，不代表当前版本已经发布。

## 对外表述建议

当你在官网、README 或其他公开材料中介绍 FileUni 时，建议统一表达为：

- FileUni 是一个轻量、可扩展、面向不同硬件档位伸缩的文件平台。
- FileUni 同时强调对 Nextcloud 客户端的兼容，覆盖文件管理、收藏、分享、媒体流程以及基于 WebDAV 的访问。
- Chat 与 Notes 属于后续开发计划，不应表述成已正式上线的功能。
