# 功能特性

本页只列出当前仓库结构、CLI/GUI 入口和已有模块能够直接证明的能力。

## CLI 与 GUI 共用核心

当前项目的 CLI 服务端和 Tauri 桌面端共用同一个 Rust 核心：

- CLI 是主服务入口
- GUI 在同一核心之上增加桌面端的服务控制、配置编辑和日志查看
- 服务器前端会静态嵌入并通过 `/ui` 提供

## 多种访问入口

当前项目已经实现的主要访问方式包括：

- `/ui` WebUI
- `/api/v1/...` HTTP API
- `/api/v1/openapi.json` OpenAPI 文档
- 启用后挂载在 `/webdav` 的 WebDAV
- 按配置启用的 S3、FTP、SFTP 服务

## 文件平台能力

仓库中已经存在专门的 VFS 存储中枢和前端模块，用于支持：

- 目录浏览、上传、下载、移动、复制、重命名、删除
- 搜索、最近访问、收藏、回收站
- WebUI 中的常见文件预览流程
- 压缩包浏览、压缩、解压等归档相关操作

部分能力是否可用，取决于具体配置和所选构建模式。

## 明确的部署模型

当前运行模型有几个很明确的约束：

- 配置来源是 `config.toml`，不是环境变量
- 运行时必须区分配置目录和应用数据目录
- 缺失必填配置时，启动会直接失败，不会静默回退到默认值
- 安装系统服务时，会持久化保存所选运行目录

## 运维能力

当前内置的运维操作包括：

- 首次初始化使用的 setup 模式
- `--configtest` 配置校验
- 系统服务的 install、uninstall、start、stop、status、reload
- 命令行重置管理员密码
- 命令行导出与导入备份

## 轻量化定位

FileUni 仍然保持轻量化文件平台的定位。公开官网当前仍在强调最低 32MB RAM 级别的部署场景，而当前工作区的结构也明显倾向于按需启用能力，而不是强制所有安装都加载全部子系统。
