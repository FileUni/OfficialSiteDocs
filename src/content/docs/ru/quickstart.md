# Быстрый старт

Это руководство основано на текущей структуре рабочей области и модели выполнения.

## 1. Выберите точку входа

В настоящее время FileUni имеет две основные точки входа:

- `fileuni` CLI: используется для запуска сервера, открытия мастера настройки при необходимости, управления сервисами и экспорта/импорта резервных копий.
- `fileuni-gui`: настольная оболочка Tauri вокруг того же базового ядра с управлением сервисом, редактированием конфигурации и тем же поведением настройки первого запуска.

Получите соответствующий пакет на [странице загрузки](https://fileuni.com/ru/download).

- Для серверного развёртывания выберите CLI-пакет.
- Для локального использования на рабочем столе выберите GUI-пакет.

## 2. Подготовьте каталог выполнения

Текущий проект теперь использует единый каталог выполнения:

- `-R` / `--runtime-dir`: единый каталог выполнения для конфигурации, install lock, базы данных, кэша и других файлов выполнения
- `--runtime-dir`: каталог выполнения только для `service install`

Фиксированный путь к файлу конфигурации:

```text
{runtime-dir}/config.toml
```

Пример структуры каталогов выполнения:

```text
./runtime
```

Для установки сервиса используйте абсолютные пути вместо относительных.

## 3. Подготовьте сервисы, на которые ссылается ваша конфигурация

FileUni не использует переменные окружения в качестве источника конфигурации. Значения выполнения должны поступать из `config.toml`.

В текущем проекте развёртывание обычно означает подготовку резервных сервисов, на которые ссылается этот файл, особенно:

- Подключение к базе данных
- Подключение к KV-сервису
- Расположения хранилища, требуемые конфигурацией VFS

Если `config.toml` или `install.lock` отсутствуют, FileUni войдёт в мастер настройки.

Мастер настройки отвечает за запись `config.toml` и `install.lock`, а также за обеспечение существования встроенной учётной записи администратора.

Обычный запуск не создаёт привилегированные учётные записи автоматически. Если учётная запись администратора отсутствует при существующем `install.lock`, запуск будет отклонён.

## 4. Запустите FileUni

Если `{runtime-dir}/install.lock` отсутствует, и CLI, и GUI откроют мастер настройки перед обычным запуском.

Если позже вы захотите снова открыть мастер настройки, удалите `{runtime-dir}/install.lock`, а затем запустите FileUni как обычно:

```bash
rm -f ./runtime/install.lock
./fileuni --runtime-dir ./runtime
```

Чтобы проверить конфигурацию без запуска полного сервера:

```bash
./fileuni --runtime-dir ./runtime config test
```

Чтобы запустить сервер обычным образом:

```bash
./fileuni --runtime-dir ./runtime
```

## 5. Откройте веб-интерфейс

После успешного запуска FileUni выводит активные адреса для:

- Веб-интерфейс: `http://<host>:<port>/`
- HTTP API: `http://<host>:<port>`
- OpenAPI JSON: `http://<host>:<port>/api/v1/openapi.json`

URL локального веб-интерфейса, используемый проектом по умолчанию:

```text
http://localhost:19000/
```

В зависимости от вашей конфигурации могут также быть включены эндпоинты S3, FTP и SFTP.
