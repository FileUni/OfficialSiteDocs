# Доступ и операции с файлами

На этой странице рассматриваются пути доступа и операции с файлами, которые могут быть проверены из текущего проекта.

## Веб-интерфейс

Встроенный фронтенд обслуживается из:

```text
/
```

Веб-интерфейс — это основная точка входа для пользователей:

- Просмотр файлов и каталогов
- Загрузка и скачивание файлов
- Потоки поиска и предварительного просмотра
- Работа с недавними элементами, избранным, корзиной и общими ресурсами
- Вызов бэкенд API из браузера

## HTTP API

Сервер предоставляет REST-стиль эндпоинты по адресу:

```text
/api/v1
```

Для проверки API и генерации клиентов проект также предоставляет:

```text
/api/v1/openapi.json
```

Это наиболее точное публичное описание текущей поверхности API.

## Доступ по протоколу

В зависимости от конфигурации FileUni может также предоставлять доступ по протоколам:

- WebDAV: монтируется по адресу `/@dav` по умолчанию (настраивается)
- S3: обслуживается на настроенном порту S3
- FTP: обслуживается на настроенном порту FTP
- SFTP: обслуживается на настроенном порту SFTP

При успешном запуске сервер выводит активный HTTP-адрес и информацию о том, включены ли S3, FTP и SFTP.

## Проверенные операции с файлами

Текущие модули VFS и фронтенда уже включают реализации для:

- Просмотр каталогов
- Загрузка файлов и обработка временных загрузок
- Скачивание файлов
- Перемещение и копирование
- Переименование
- Удаление в корзину и восстановление из корзины
- Поиск
- Избранное
- Записи о ресурсах и фильтры ресурсов
- Просмотр архивов, сжатие и распаковка

## Операционные примечания

Некоторые из этих возможностей ограничены конфигурацией. На практике:

- Если протокол отключён в конфигурации, он будет недоступен во время выполнения.
- Если опциональная возможность фронтенда выключена, соответствующая запись может исчезнуть из UI.
- Режимы сборки также могут влиять на то, что включается в пакет.

## Рекомендуемые варианты доступа

Используйте веб-интерфейс, когда вам нужно:

- Самый простой способ просмотра и управления файлами
- Потоки предварительного просмотра
- Административные операции, уже предоставленные фронтендом

Используйте WebDAV, S3, FTP или SFTP, когда вам нужно:

- Интеграция с существующими инструментами
- Монтирование дисков или сторонние файловые менеджеры
- Автоматизированные рабочие процессы и скрипты
