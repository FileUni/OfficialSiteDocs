# Внешние зависимости

FileUni использует небольшой набор внешних исполняемых файлов для функций предварительного просмотра/миниатюр/сжатия, а также опциональные KV и SQL сервисы для продакшн-развёртываний. На этой странице объясняется, почему эти зависимости существуют, когда они нужны, как их установить и какие элементы конфигурации необходимо настроить после этого.

## 1. Внешние исполняемые файлы

Эти инструменты обеспечивают функции предварительного просмотра, миниатюр и сжатия. Если вам не нужна функция, вы можете пропустить соответствующий инструмент, но убедитесь, что связанные элементы конфигурации соответствующим образом настроены.

- 7-Zip (`7z` / `7z-full`)
  - Назначение: Форматы высокого сжатия и многопоточное сжатие/распаковка.
  - Нужен когда: Вам нужна поддержка формата 7z или ускоренное сжатие сверх нативного ZIP/TAR.
- libvips
  - Назначение: Быстрый рендеринг миниатюр изображений/PDF.
  - Нужен когда: Вы включаете миниатюры изображений или PDF (предпочтительный движок).
- ImageMagick
  - Назначение: Резервный рендеринг миниатюр и генерация текстовых миниатюр.
  - Нужен когда: Вы включаете текстовые миниатюры или хотите резервный вариант для миниатюр PDF/изображений.
- FFmpeg
  - Назначение: Видеоминиатюры (извлечение кадров) и метаданные видео.
  - Нужен когда: Вы включаете видеоминиатюры.
- LibreOffice (`soffice`)
  - Назначение: Миниатюры офисных документов (конвертация в PDF, затем миниатюризация).
  - Нужен когда: Вы включаете миниатюры офисных документов.
- Инструментарий LaTeX (`latexmk` + `xelatex`)
  - Назначение: Предпросмотр LaTeX и миниатюры LaTeX (компиляция в PDF).
  - Нужен когда: Вы включаете предпросмотр LaTeX или миниатюры LaTeX.

## 2. Опциональные внешние сервисы (KV и SQL)

Эти сервисы опциональны и выбираются конфигурацией. Они не включены в Docker-образ и должны развёртываться отдельно при необходимости.

- KeyDB / Redis / Valkey
  - Назначение: Распределённый KV-кэш и координация.
  - Нужен когда: Вы выбираете KV-тип кэша для продакшна или мультиинстансных развёртываний.
- PostgreSQL (pgsql)
  - Назначение: Основная SQL база данных для продакшна.
  - Нужен когда: Вы выбираете `postgres`/`pgsql` как тип базы данных.
- SQLite
  - Назначение: Встроенная SQL база данных для одиночных узлов или сред с ограниченными ресурсами.
  - Нужен когда: Вы выбираете `sqlite` как тип базы данных.

## 3. Политика Docker-пакетирования

Стандартный Docker-образ включает только лёгкие исполняемые файлы, необходимые для предпросмотра/миниатюр/сжатия. Он не включает LibreOffice (слишком большой) и какие-либо KV/SQL сервисы (KeyDB/Redis/Valkey/PostgreSQL), так как они опциональны и должны работать как внешние сервисы.

## 4. Элементы конфигурации для обновления после установки

Внешние исполняемые файлы:
- `thumbnail.tools.vips_path`
- `thumbnail.tools.imagemagick_path`
- `thumbnail.tools.ffmpeg_path`
- `thumbnail.tools.libreoffice_path`
- `latex_preview.latexmk_path`
- `file_compress.exe_7zip_path`

Переключатели функций и ограничения (обычно используются с вышеуказанными инструментами):
- `latex_preview.enable_latexmk`
- `thumbnail.enabled`
- `thumbnail.<image|video|pdf|office|text>.enabled`

Опциональные KV-сервисы:
- `fast_kv_storage_hub.kv_type`
- `fast_kv_storage_hub.redis_url`

Опциональные SQL-сервисы:
- `database.db_type`
- `database.postgres_config.database_dsn`
- `database.sqlite_config.database_dsn`

## 5. Справочник команд ручной установки

Debian / Ubuntu (внешние исполняемые файлы):

```bash
sudo apt-get update
sudo apt-get install -y \
  ffmpeg \
  imagemagick \
  libvips-tools \
  p7zip-full \
  latexmk \
  texlive-xetex
```

LibreOffice (только если включены миниатюры Office):

```bash
sudo apt-get install -y libreoffice
```

Опциональные сервисы через Docker (рекомендуется для быстрой настройки):

```bash
# Redis
docker run -d --name redis -p 6379:6379 redis:7

# Valkey
docker run -d --name valkey -p 6379:6379 valkey/valkey:7

# KeyDB
docker run -d --name keydb -p 6379:6379 eqalpha/keydb:alpine

# PostgreSQL
docker run -d --name postgres -p 5432:5432 \
  -e POSTGRES_PASSWORD=admin888 \
  -e POSTGRES_DB=fileuni \
  postgres:16
```

После установки убедитесь, что пути к исполняемым файлам и URL сервисов в `config.toml` соответствуют вашей среде.
