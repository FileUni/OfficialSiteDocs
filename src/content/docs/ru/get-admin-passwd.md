# Сброс пароля администратора

Если вы потеряли встроенный пароль администратора, поддерживаемый путь восстановления больше не является специальным флагом сброса CLI.

FileUni теперь рассматривает `{runtime-dir}/install.lock` как маркер завершения установки:

- Если `install.lock` существует, FileUni запускается обычным образом.
- Если `install.lock` отсутствует, и CLI, и GUI блокируют обычный запуск и открывают центр настроек.
- Завершение центра настроек снова записывает `install.lock` и позволяет системе продолжить запуск.

Это означает, что восстановление пароля администратора теперь выполняется путём повторного открытия центра настроек.

## Шаги восстановления

1. Остановите запущенный сервис FileUni или настольный экземпляр.
2. Найдите ваш каталог выполнения.
3. Удалите `{runtime-dir}/install.lock`.
4. Запустите FileUni снова из CLI или GUI.
5. FileUni автоматически откроет центр настроек.
6. В центре настроек установите новый пароль администратора и завершите первичную настройку.

## Пример

Если ваш каталог выполнения — `/srv/fileuni`, удалите этот файл:

```bash
rm /srv/fileuni/install.lock
```

Затем перезапустите FileUni:

```bash
fileuni --runtime-dir /srv/fileuni
```

Или снова откройте настольное приложение и выберите тот же каталог выполнения.

## Важные примечания

- Удаление `install.lock` и перезапуск фактически являются точкой входа для сброса системы для этого развёртывания.
- Это само по себе не удаляет существующую базу данных или данные, но заставляет вас снова пройти через поток инициализации.
- Используйте тот же каталог выполнения, что и у развёртывания, которое вы восстанавливаете.
- Если вы укажете другой каталог выполнения, можно по ошибке инициализировать другое развёртывание.

## Устранение неполадок

### Центр настроек не открылся

Проверьте следующие пункты:

- Вы удалили правильный файл: `{runtime-dir}/install.lock`
- Вы перезапустили то же развёртывание
- Каталог выполнения, переданный через `-R/--runtime-dir`, указан верно

### Я не знаю свой каталог выполнения

Вы можете восстановить его из:

- Вашей команды установки сервиса или менеджера сервисов
- Вашего выбора каталога выполнения в настольном приложении
- Вашего скрипта запуска или истории оболочки
