# Сброс пароля администратора

Если вы потеряли встроенный пароль администратора, поддерживаемый путь восстановления больше не является специальным флагом сброса CLI.

FileUni теперь рассматривает `{config-date}/install.lock` как маркер завершения установки:

- Если `install.lock` существует, FileUni запускается обычным образом.
- Если `install.lock` отсутствует, и CLI, и GUI блокируют обычный запуск и принудительно запускают мастер настройки.
- Завершение мастера настройки снова записывает `install.lock` и позволяет системе продолжить запуск.

Это означает, что восстановление пароля администратора теперь выполняется путём повторного открытия мастера настройки.

## Шаги восстановления

1. Остановите запущенный сервис FileUni или настольный экземпляр.
2. Найдите ваш каталог конфигурации.
3. Удалите `{config-date}/install.lock`.
4. Запустите FileUni снова из CLI или GUI.
5. FileUni автоматически войдёт в мастер настройки.
6. В мастере настройки установите новый пароль администратора и завершите настройку.

## Пример

Если ваш каталог конфигурации — `/etc/fileuni`, удалите этот файл:

```bash
rm /etc/fileuni/install.lock
```

Затем перезапустите FileUni:

```bash
fileuni -c /etc/fileuni -A /var/lib/fileuni
```

Или снова откройте настольное приложение и выберите те же каталоги выполнения.

## Важные примечания

- Удаление `install.lock` и перезапуск фактически являются точкой входа для сброса системы для этого развёртывания.
- Это не удаляет вашу существующую базу данных или данные приложения само по себе, но заставляет вас пройти через поток инициализации заново.
- Используйте тот же каталог конфигурации и каталог данных приложения, что и развёртывание, которое вы восстанавливаете.
- Если вы укажете другую пару каталогов выполнения, вы можете по ошибке инициализировать другое развёртывание.

## Устранение неполадок

### Мастер настройки не появился

Проверьте следующие пункты:

- Вы удалили правильный файл: `{config-date}/install.lock`
- Вы перезапустили то же развёртывание
- Каталоги выполнения, переданные через `-c/--config-date` и `-A/--AppDataDir`, верны

### Я не знаю мой каталог конфигурации

Вы можете восстановить его из:

- Вашей команды установки сервиса или менеджера сервисов
- Вашего выбора каталога выполнения в настольном приложении
- Вашего скрипта запуска или истории оболочки
