# アクセスとファイル操作

このページでは、現在のプロジェクトから検証できるアクセスパスとファイル操作に焦点を当てます。

## Web UI

埋め込みフロントエンドは以下から提供されます：

```text
/
```

Web UI は以下のメインの人間向けエントリポイントです：

- ファイルとディレクトリの参照
- ファイルのアップロードとダウンロード
- 検索とプレビューフロー
- 最近のアイテム、お気に入り、ゴミ箱、共有の操作
- ブラウザフロントエンドからバックエンド API の呼び出し

## HTTP API

サーバーは以下で REST スタイルのエンドポイントを公開しています：

```text
/api/v1
```

API の検査とクライアント生成のために、プロジェクトは以下も公開しています：

```text
/api/v1/openapi.json
```

これは現在マウントされている API サーフェスの最も正確な公開記述です。

## プロトコルアクセス

設定に応じて、FileUni はプロトコルベースのアクセスも公開できます：

- WebDAV：デフォルトで `/@dav` にマウント（設定可能）
- S3：設定された S3 ポートで提供
- FTP：設定された FTP ポートで提供
- SFTP：設定された SFTP ポートで提供

サーバーが正常に起動すると、アクティブな HTTP アドレスと、S3、FTP、SFTP が有効かどうかを出力します。

## 検証済みのファイル操作

現在の VFS とフロントエンドモジュールには、以下の実装が既に含まれています：

- ディレクトリの参照
- ファイルのアップロードと一時アップロード処理
- ファイルのダウンロード
- 移動とコピー
- 名前変更
- ゴミ箱への削除とゴミ箱からの復元
- 検索
- お気に入り
- 共有レコードと共有フィルター
- アーカイブの参照、圧縮、展開

## 運用上の注意

これらの機能の一部は設定によってフィーチャーゲートされています。実際には：

- プロトコルが設定で無効な場合、ランタイムで利用できません。
- オプションのフロントエンド機能がオフの場合、エントリが UI から消える可能性があります。
- ビルドモードもバンドルされる内容に影響する可能性があります。

## 推奨されるアクセス選択

以下の場合に Web UI を使用：

- ファイルを参照・管理する最も簡単な方法が必要
- プレビューフロー
- フロントエンドで既に公開されている管理操作

以下の場合に WebDAV、S3、FTP、SFTP を使用：

- 既存のツールとの統合
- ドライブのマウントやサードパーティのファイルマネージャー
- 自動化されたワークフローとスクリプト
