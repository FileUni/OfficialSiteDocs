# 機能

このページでは、現在のリポジトリとランタイムエントリポイントから検証できる機能を一覧表示します。

## CLI と GUI の共有コア

CLI サーバーと Tauri デスクトップアプリは同じ Rust コアライブラリを使用しています：

- CLI はメインサーバーのエントリポイントです。
- GUI は同じバックエンドをデスクトップネイティブのサービス制御、設定編集、ログ表示でラップしています。
- サーバーのフロントエンドは静的に埋め込まれ、`/` から提供されます。

## 複数のアクセスパス

現在のプロジェクトは、同じファイルプラットフォームに到達するいくつかの方法を公開しています：

- `/` で Web インターフェース
- `/api/v1/...` で HTTP API
- `/api/v1/openapi.json` で OpenAPI ドキュメント
- デフォルトで `/@dav` にマウントされる WebDAV（設定可能）
- 設定で有効な場合、S3、FTP、SFTP サービス

## ファイルプラットフォームの機能

ワークスペースには、専用の VFS ストレージハブと以下のフロントエンドモジュールが含まれています：

- ディレクトリの参照、アップロード、ダウンロード、移動、コピー、名前変更、削除
- 検索、最近の履歴、お気に入り、ゴミ箱ビュー
- Web インターフェースでの一般的なファイルタイプのファイルプレビューフロー
- アーカイブの参照、圧縮、展開などのアーカイブ関連操作

一部の機能の正確な可用性は、設定と選択したビルドモードによって異なります。

## デプロイモデル

現在のランタイムモデルは意図的に明示的です：

- 設定は環境変数ではなく `config.toml` から取得されます
- ランタイムディレクトリは設定パスとアプリデータパスに分離されています
- 必須の設定フィールドが欠けている場合、サイレントにデフォルトにフォールバックするのではなく、起動が拒否されます
- サービスインストールは選択されたランタイムディレクトリを永続化します

## 運用とメンテナンス

現在の組み込みメンテナンスアクションには以下が含まれます：

- 初回実行設定のためのセットアップモード
- `--configtest` での設定検証
- システムサービスのインストール、アンインストール、開始、停止、ステータス、リロード
- コマンドラインからの管理者パスワードリセット
- コマンドラインからのバックアップエクスポートとインポート

## 軽量なポジショニング

FileUni は依然として軽量なファイルプラットフォームとして位置づけられています。公開サイトでは現在、32 MB RAM という低いデプロイが説明されており、ワークスペース自体はすべてのサブシステムをすべてのインストールで強制的に有効にするのではなく、選択的にサービスを有効にするように構成されています。
