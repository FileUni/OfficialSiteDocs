# Nextcloud 互換性

このページでは、FileUni を Nextcloud クライアントから利用する際の互換性目標を説明します。

低性能ハードウェアで動作し、高性能ハードウェアではより高い性能を引き出せることに加えて、Nextcloud クライアントとの互換性は FileUni が外部向け資料で明確に打ち出すべき重要な特長の一つです。

## 互換性の目標

FileUni は、Nextcloud ベース環境から移行するユーザーが期待する主要な Nextcloud クライアントの利用フローとの互換性を保つことを目指しています。

現在の重点は次の通りです。

- WebDAV 経由のファイル同期とリモートアクセス
- デスクトップおよびモバイルクライアントで使われるファイル管理操作
- お気に入り、共有、メディア関連フロー、一般的なメタデータ操作
- 既存の Nextcloud クライアント利用習慣をできるだけ維持すること

## 現在の対象範囲

現在のリポジトリには、この方向性に必要な基盤がすでに含まれています。

- ストレージスタック内の WebDAV サービス対応
- 閲覧、アップロード、ダウンロード、名前変更、移動、コピー、削除、最近使った項目、お気に入り、ごみ箱を扱う Web UI と API フロー
- フロントエンドにおけるメディアプレビューと一般的なファイルプレビュー
- 展開規模に応じて有効化できるモジュラーなプロトコルサービス

これらは Nextcloud 互換性作業の土台です。実際のクライアント挙動は、設定、有効化されたモジュール、利用する Nextcloud クライアントのバージョンによって変わります。

## チェックリスト

| 項目 | 目標 | 状態 |
| --- | --- | --- |
| WebDAV エンドポイントアクセス | Nextcloud クライアントの WebDAV 接続フローと互換 | 基盤あり |
| ファイル管理 | 閲覧、アップロード、ダウンロード、名前変更、移動、コピー、削除 | 現行プラットフォームで実装済み |
| お気に入り | お気に入り関連フローの互換性を保つ | 実装済みの方向性あり |
| 共有 | 共有関連フローの互換性を保つ | 互換性目標 |
| メディア | メディア閲覧とプレビューを使いやすく保つ | 実装済みの方向性あり |
| Chat | 今後の追加機能 | 計画中 |
| Notes | 今後の追加機能 | 計画中 |

## 状態ラベルの意味

- 基盤あり: この領域に必要なプロトコル層またはプラットフォーム層がすでにあります。
- 現行プラットフォームで実装済み: FileUni 自体にその機能がすでにあります。
- 実装済みの方向性あり: 関連機能はすでに存在し、互換性作業はその上に構築されます。
- 互換性目標: 明示された製品方針の一部であり、すべてのクライアントの細かなケースがすでに完了していることを意味しません。
- 計画中: 将来の範囲であり、現行機能ではありません。

## 外部向け表現の指針

- FileUni は軽量でスケーラブルなファイルプラットフォームです。
- FileUni は、ファイル管理、お気に入り、共有、メディアフロー、WebDAV ベースのアクセスにおいて Nextcloud クライアントとの互換性維持も重視しています。
- Chat と Notes は後続ロードマップであり、すでに提供済みの機能として表現すべきではありません。
