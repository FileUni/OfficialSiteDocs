# 管理者パスワードのリセット

組み込み管理者パスワードを紛失した場合、サポートされる回復パスはもはや専用の CLI リセットフラグではありません。

FileUni は現在、`{config-date}/install.lock` をインストール完了マーカーとして扱います：

- `install.lock` が存在する場合、FileUni は通常どおり起動します。
- `install.lock` がない場合、CLI と GUI の両方が通常の起動をブロックし、セットアップウィザードを強制します。
- セットアップウィザードを完了すると、`install.lock` が再び書き込まれ、システムは起動を継続できます。

つまり、管理者パスワードの回復はセットアップウィザードを再度開くことで行います。

## 回復手順

1. 実行中の FileUni サービスまたはデスクトップインスタンスを停止します。
2. 設定ディレクトリを見つけます。
3. `{config-date}/install.lock` を削除します。
4. CLI または GUI から FileUni を再度起動します。
5. FileUni は自動的にセットアップウィザードに入ります。
6. セットアップウィザードで、新しい管理者パスワードを設定し、セットアップを完了します。

## 例

設定ディレクトリが `/etc/fileuni` の場合、このファイルを削除します：

```bash
rm /etc/fileuni/install.lock
```

その後、FileUni を再起動します：

```bash
fileuni -c /etc/fileuni -A /var/lib/fileuni
```

または、デスクトップアプリを再度開き、同じランタイムディレクトリを選択します。

## 重要な注意事項

- `install.lock` を削除して再起動することは、事実上そのデプロイのシステムリセットエントリです。
- これは既存のデータベースやアプリデータ自体を削除しませんが、初期化フローに戻すことを強制します。
- 回復しているデプロイと同じ設定ディレクトリとアプリデータディレクトリを使用してください。
- 異なるランタイムディレクトリペアを指定すると、誤って異なるデプロイを初期化する可能性があります。

## トラブルシューティング

### セットアップウィザードが表示されない

以下の項目を確認してください：

- 正しいファイルを削除しました：`{config-date}/install.lock`
- 同じデプロイを再起動しました
- `-c/--config-date` と `-A/--AppDataDir` で渡されたランタイムディレクトリが正しい

### 設定ディレクトリがわからない

以下から回復できます：

- サービスインストールコマンドまたはサービスマネージャー
- デスクトップアプリのランタイムディレクトリ選択
- スタートアップスクリプトまたはシェル履歴
