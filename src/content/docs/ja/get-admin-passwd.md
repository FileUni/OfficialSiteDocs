# 管理者パスワードのリセット

組み込み管理者パスワードを紛失した場合、サポートされる回復パスはもはや専用の CLI リセットフラグではありません。

FileUni は現在、`{runtime-dir}/install.lock` をインストール完了マーカーとして扱います：

- `install.lock` が存在する場合、FileUni は通常どおり起動します。
- `install.lock` がない場合、CLI と GUI の両方が通常の起動をブロックし、設定センターを開きます。
- 設定センターを完了すると、`install.lock` が再び書き込まれ、システムは起動を継続できます。

つまり、管理者パスワードの回復は設定センターを再度開くことで行います。

## 回復手順

1. 実行中の FileUni サービスまたはデスクトップインスタンスを停止します。
2. ランタイムディレクトリを見つけます。
3. `{runtime-dir}/install.lock` を削除します。
4. CLI または GUI から FileUni を再度起動します。
5. FileUni は自動的に設定センターを開きます。
6. 設定センターで新しい管理者パスワードを設定し、初回設定を完了します。

## 例

ランタイムディレクトリが `/srv/fileuni` の場合、このファイルを削除します：

```bash
rm /srv/fileuni/install.lock
```

その後、FileUni を再起動します：

```bash
fileuni --runtime-dir /srv/fileuni
```

または、デスクトップアプリを再度開き、同じランタイムディレクトリを選択します。

## 重要な注意事項

- `install.lock` を削除して再起動することは、事実上そのデプロイのシステムリセットエントリです。
- これは既存のデータベースやデータ自体を削除しませんが、初期化フローに戻すことを強制します。
- 回復しているデプロイと同じランタイムディレクトリを使用してください。
- 異なるランタイムディレクトリを指定すると、誤って別のデプロイを初期化する可能性があります。

## トラブルシューティング

### 設定センターが表示されない

以下の項目を確認してください：

- 正しいファイルを削除しました：`{runtime-dir}/install.lock`
- 同じデプロイを再起動しました
- `-R/--runtime-dir` で渡したランタイムディレクトリが正しい

### ランタイムディレクトリがわからない

以下から回復できます：

- サービスインストールコマンドまたはサービスマネージャー
- デスクトップアプリのランタイムディレクトリ選択
- スタートアップスクリプトまたはシェル履歴
