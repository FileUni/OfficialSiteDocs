# 訪問方式與檔案操作

本頁只描述當前專案裡已經能從原始碼和入口層確認的訪問方式與檔案操作。

## WebUI

內嵌前端透過下面的路徑提供：

```text
/
```

WebUI 是當前最主要的人機互動入口，負責：

- 瀏覽檔案與目錄
- 上傳、下載
- 搜尋與預覽
- 最近訪問、收藏、回收站、分享等檢視
- 在瀏覽器中呼叫後端 API

## HTTP API

服務端的 REST 風格介面位於：

```text
/api/v1
```

如果要檢視介面定義或生成客戶端型別，當前專案還會暴露：

```text
/api/v1/openapi.json
```

這是當前對外最準確的介面面描述。

## 協議訪問

按設定啟用後，FileUni 還可以提供這些標準協議入口：

- WebDAV：預設掛載在 `/@dav`（可設定）
- S3：監聽設定中指定的 S3 埠
- FTP：監聽設定中指定的 FTP 埠
- SFTP：監聽設定中指定的 SFTP 埠

服務啟動成功後，終端會列印當前 HTTP 地址，以及 S3、FTP、SFTP 是否啟用。

## 已確認存在的檔案操作

當前 VFS 與前端模組裡已經有實現的能力包括：

- 目錄瀏覽
- 檔案上傳與臨時上傳處理
- 檔案下載
- 移動與複製
- 重新命名
- 刪除到回收站以及從回收站恢復
- 搜尋
- 收藏
- 分享記錄與分享篩選
- 壓縮包瀏覽、壓縮、解壓

## 使用上的說明

這些能力裡有一部分會受到設定開關控制。實際部署時：

- 如果協議在設定中被關閉，執行時就不會開放
- 如果某些前端能力被關閉，介面入口也可能不會顯示
- 不同 build mode 也可能影響最終打包內容

## 推薦的訪問選擇

以下場景優先使用 WebUI：

- 想要最直接地瀏覽和管理檔案
- 需要預覽能力
- 需要透過現成前端完成管理操作

以下場景更適合使用 WebDAV、S3、FTP、SFTP：

- 需要接入現有工具鏈
- 需要掛載磁碟或接第三方檔案管理器
- 需要指令碼化和自動化訪問
